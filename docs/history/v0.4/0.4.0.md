## [v0.4.0] - 2026-07-24

### 技术改进/重构 (Changed/Refactor)

* 将内部项目身份从 `RandomLoadout` 统一迁移为 `EtgGameplayDashboard`，同步更新解决方案、程序集、命名空间、源码目录、测试目录、配置文件、插件 GUID 和构建/发布工具。
* 将运行时数据文件统一迁移为 `EtgGameplayDashboard.*` 命名，并更新默认配置、部署路径、文档路径和日志工具命令。
* 将拾取物信息(nearby-pickup info)统计数据升级为 `schemaVersion: 3`，把 `stats[*].value` 重构为不可兼容的 `stats[*].parts[*]` 结构，支持独立的数值和标签显示。
* 更新物品信息读取器、统计数据渲染器、数据生成器、翻译工作流和统计值扫描器以使用新的 `parts` 结构。
* 为同名的 `Master Round` 物品补充 `[I]`、`[II]`、`[III]`、`[IV]`、`[V]` 英文区分标记，避免不同楼层大师弹无法准确识别。
* 扩展拾取物统计标签本地化，支持 `Large`、`Medium`、`Small`、`Droplet`、`Droplets`、`Varies heavily` 等复合统计标签的中文显示。
* 从游戏资源提取协同效果(Synergies)的中文命名，写入 `tools/data/reference/etg-synergies.json`，并用游戏内中文命名替换旧的临时中文协同翻译。
* 更新仓库规则、当前文档、日志前缀和日志提取工具，统一使用 `EtgGameplayDashboard` 项目身份。

### 修复 (Fixed)

* 修复 `Wind Up Gun` 的 `Breakfast Club` 和 `Wound Up` 协同效果被错误放入备注(notes)的问题。
* 修复 `Dueling Laser` 的 `Banshee Cry` 和 `Death Square` 协同效果被错误放入备注(notes)的问题。
* 修复物品信息统计值包含多个英文标签时无法本地化的问题，例如 `Large`、`Medium`、`Small` 和 `Droplets`。
* 修复中文统计标签扫描器无法识别结构化统计部分(parts)的问题，并将未知标签扫描结果降为 0 项。

### 弃用/删除 (Deprecated/Removed)

* 删除旧的 `RandomLoadout` 项目身份文件名、源码路径、程序集名称和插件配置身份；本版本不提供旧命名兼容读取。
* 删除旧的 `stats[*].value` 统计数据格式，统一使用 `stats[*].parts`。
