## [v0.3.6] - 2026-07-06

### 新增功能 (Added)
* 新增游戏启动窗口焦点恢复服务 `GameWindowFocusService`，支持在 Steam 启动游戏且窗口处于后台或最小化时，通过 Win32 API 自动定位并置顶 BepInEx 控制台与游戏窗口，同时添加了 `EnableStartupWindowFocusVerboseLogs` 调试开关及说明文档 `docs/operations/startup-window-focus.md`。
* 新增性能诊断模块 `PerformanceDiagnostics` 与 `Plugin.Performance.cs`，通过滚动 FPS 统计、长帧耗时警告（默认 40ms 阈值）、`Plugin.Update()` 分步更新子任务耗时追踪，监控模组加载导致的运行卡顿，同时添加了 `EnablePerformanceVerboseLogs` 调试开关及说明文档 `docs/operations/logging-performance.md`。
* 新增自动扫描校验源文件许可证头部声明的 `tools/devtools/check_license_headers.py` 脚本，并将其集成至 MSBuild 编译流程 `build/LicenseHeaderCheck.targets` 中，保证每次项目编译前自动检测头部声明合规性。
* 新增性能及窗口焦点相关配置项 `EnableNearbyPickupVerboseLogs`、`EnableStartupWindowFocusVerboseLogs` 和 `EnablePerformanceVerboseLogs`。

### 技术改进/重构 (Changed/Refactor)
* 重构项目开源授权协议，将整体许可证从 `MIT` 变更为 `GPL-3.0-only`，批量为所有源文件添加版权许可证声明头部，并同步更新了发布包构建工具中自动生成的第三方通知许可声明元数据。
* 重构靠近拾取物提示服务 `NearbyPickupTipService`，将其由原先每 0.15 秒定时轮询房间掉落物的设计重构为基于 Harmony 钩子事件驱动的被动响应机制（覆盖 `Gun`、`PassiveItem`、`PlayerItem`、`ShopItemController` 和 `RewardPedestal` 的进入/离开/交互事件），大幅降低物品密集或商店场景下的 CPU 额外开销，并新增说明文档 `docs/operations/logging-nearby-pickup.md`。
* 优化 `FoyerCharacterSwitchService` 角色切换服务，支持在性能日志开启时，精确统计并输出在 `裂隙(Breach)` 中执行角色强制切换的耗时指标。

### 修复 (Fixed)
* 修复在 `JsonPickupGameplayProvider.cs` 和 `Plugin.PickupWikiTips.cs` 中对于 `拾取物(pickups)` ID 范围判定的逻辑漏洞，允许 ID 为 `0` 的物品作为有效物品进行图鉴信息匹配与渲染（解决“阿拉丁神灯 (Magic Lamp)”这一 ID 为 0 的拾取物无法在靠近时显示图鉴信息的问题）。
* 修复 `NearbyPickupTipHooks` 钩子中 `ShopItemController` 和 `RewardPedestal` 的 `OnEnteredRange` 方法的 Harmony 参数绑定名称限制，确保其参数名与 ETG 原版签名中的 `interactor` 保持一致，解决原先因参数名不匹配导致钩子在运行时拦截失效的异常。
