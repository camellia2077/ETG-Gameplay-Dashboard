## [v0.3.7] - 2026-07-10

### 技术改进/重构 (Changed/Refactor)
* 重构 `靠近物品信息显示(nearby pickup info)` 的数据加载架构为 Schema V2：为了优化 JSON 结构而进行了不兼容的更新，升级为全新的 Schema V2。将原本分散在 `RandomLoadout.pickup-gameplay.en.json` (39,506 行) 与 `RandomLoadout.pickup-gameplay.zh-CN.work.json` (45,079 行) 且包含翻译的旧版文件（共 84,585 行），合并为按 `pickupId` 存储基础 `物品(items)` 信息的单份 `defaults/catalog/RandomLoadout.pickup-gameplay.json` 文件（共 58,549 行）；同时将原本混杂其中的公用界面区块标签（`sections`）、属性键名（`stats`）与属性显示共用译文（`displayValues`，涵盖品质等级、物品类型与状态条件等）抽取为独立的 `defaults/catalog/RandomLoadout.pickup-info-terms.json` 翻译字典文件（共 890 行）。重构解耦后：
  - 两份新文件总行数降至 59,439 行，累计精简了 25,146 行（降幅约 29.73%）；
  - JSON 属性键值（key）总数从 49,692 个减少至 29,405 个，累计精简了 20,287 个键（降幅约 40.83%），显著降低了运行时内存占用与文件解析开销，实现了物品描述数据集的结构化解耦与加载优化。
* 引入 Schema V2 字段约定说明文档 `docs/reference/pickup-gameplay-schema-v2.md` 与数据构建脚本 `tools/data/build_pickup_gameplay_v2.py`，以规范和辅助编译生成新版物品数据。
* 补全 `靠近物品信息显示(nearby pickup info)` 链路的诊断日志，复用已有的 `EnableNearbyPickupVerboseLogs` 调试开关，默认不输出日志以避免刷屏，方便开发与排错。
* 重构 JSON/JSON5 读取逻辑：将原本使用正则表达式提取部分字段的旧口径，替换为引入仓库已有 `Newtonsoft.Json.dll` 库进行结构化对象解析的全新口径，以兼容项目的 .NET Framework 3.5 运行环境，并显著提升文件读取速度与解析健壮性；此改动覆盖了物品 gameplay 信息、UI 术语(terms)、`别名(alias)`、`开局物品(start items)` `预设(preset)`、`规则(rules)`、`随机池(random pool)` 选择状态及中英文`本地化(localization)`文件的读取，同时保留 `Json5TextNormalizer` 以继续支持注释和尾逗号。
* 调整 `控制面板(control panel)` 的主界面与 `设置(settings)` 页面界面：
  - 更新主界面的开启与关闭提示信息：将原本静态且表意模糊的键盘打开提示，升级为动态刷新口径，根据用户实际绑定的自定义按键实时显示当前的 `键盘(keyboard)` 快捷键（如 `F7`、`F8` 等）以及固定的 360 `手柄(controller)` R3 按键。
  - 将 `设置(settings)` 页面中 `手柄说明(Controller Help)`、`键盘说明(Keyboard Help)`、`高级工具(Advanced Tools)` 选项右侧的按钮文本由 `Change / 切换` 修改为 `View Details / 查看详情`（保留其他用于切换设置按钮的 `Change / 切换` 文本）：由于这三项并非用于切换选项的设置项，而是用于进入展示说明与工具信息，因而优化文本表述以符合其真实操作行为。
  - 更新 `设置(settings)` 页面底部的项目声明文本，推荐从 GitHub 官方渠道下载最新包并查看每次 Release 的更新说明，防范可能包含广告或被恶意篡改的第三方打包版本。
* 调整英文 Pickups 页面标题：由 `Pickup Browser` 变更为 `Items`，以解决此前与 General 分类下 `Pickups` 词汇重复的问题（中文标题“物品”保持未变）。
* 更新翻译流程及相关脚本与说明文档（包括 `tools/translate/AGENTS.md`、`tools/translate/README.md`、`tools/translate/MUST.md`、`tools/translate/SCRIPTS.md`）：由于物品信息 JSON 数据结构变更为 Schema V2，导致原先针对 V1 版本设计的翻译与检查流程失效，因此将整体翻译工作流、校验脚本和相关说明文档全面更新以与 Schema V2 保持一致，并同步修改日志、部署及配置格式等文档中对 nearby pickup/v2 的引用口径。
* 优化单元测试用例，补齐针对 `别名(alias)`、`预设(preset)` `规则(rules)` 及 `随机池(random pool)` 选择状态等 JSON5 读取场景的测试，并修正规则文件测试以适配当前的 preset 目录加载架构，确保 36 项测试全部通过。

### 修复 (Fixed)
* 修复因引入 Newtonsoft.Json 重构而导致 `pickups` 字典对象中纯数字键（如 `"0"`、`"541"`）解析失效的回归 Bug（该问题曾导致加载零条目并使 `靠近物品信息显示(nearby pickup info)` 功能失效）。
* 修复 `设置(settings)` 页面底部项目说明文本被截断的问题，启用自动换行并将区域高度从 20 像素增至 72 像素。

### 弃用/删除 (Deprecated/Removed)
* 移除 `defaults/catalog/` 目录下的旧版翻译源文件 `RandomLoadout.pickup-gameplay.en.json` 与 `RandomLoadout.pickup-gameplay.zh-CN.work.json`，并将其迁移至 `defaults/catalog/legacy/` 目录下。
* 迁移 `tools/` 默认路径和文档中的相关引用路径到 `legacy/`，使 `build_pickup_gameplay_v2.py` 默认从 `legacy/` 目录读取源文件。
