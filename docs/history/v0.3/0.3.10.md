## [v0.3.10] - 2026-07-23

* 新增键盘(keyboard)辅助自瞄功能：在 `src/RandomLoadout/Commands/KeyboardAimAssistService.cs` 中实现键鼠辅助自瞄，提供 `普通自动瞄准` (`Auto Aim`，基础角度 15°) 与 `超级自动瞄准` (`Super Auto Aim`，基础角度 25°) 两种模式（名称直接沿用游戏内原版手柄辅助瞄准设置面板的官方中英文名称），并在开启或切换模式时于结果描述中实时显示对应的倾角角度提示（如 `普通自动瞄准（基础角度 15°）`）；同时支持 `0.5x` 至 `2.0x` 的自瞄倍率调节（实际有效吸附角度为 `基础角度 × 倍率`）。实现上通过拦截 `DetermineAimPointInWorld` 结合弹道飞行时间预判(`leadTime`)与物理碰撞射线检测(`Raycast`)过滤隔墙目标，并在 `src/RandomLoadout.Core/Cursor/` 与 `Input/` 中引入光标渲染与输入所有权管理模块。配套新增了 `docs/reference/controller-aim-lock.md` 使用参考与 `docs/operations/logging-controller-aim.md` 调试诊断文档。
* 新增手柄(controller)视角固定功能：在 `src/RandomLoadout/Commands/ControllerAimLockService.cs` 中实现手柄视角固定(`Controller Aim Lock`)。开启该功能后，当玩家推动手柄右摇杆(`R3`)转动瞄准方向时，自动拦截并抑制原版 `CameraController` 的瞄准偏移量(`aim-look`)，使游戏视角镜头始终平稳居中固定在玩家角色身上，而不随着 `R3` 摇杆的转动产生画面偏转与晃动。
* 新增 BOSS 选择与 BOSS 战斗房间的选择功能：在控制面板(control panel)的房间(`Room`)菜单中新增 BOSS 选择子页面 (`RoomDebugCommandService.BossSelection.cs`)，支持提前选择下一层关卡将要生成的 BOSS 种类（在大厅选择时作用于第 1 层关卡，局内选择时作用于下一层关卡）。设计上通过修改地牢生成前的 `BossManager.PriorFloorSelectedBossRoom` 预设，在生成下一层地图时直接载入对应 BOSS，而非在当前已生成的地图上强行替换 BOSS 房间（避免因不同 BOSS 战场房间形状与布局原型存在差异而导致破坏性的地图墙体错位与碰撞体撕裂）；当选定的 BOSS 包含多种原版房间地形时，面板提供第二行不同 BOSS 战斗房间布局原型(`PrototypeDungeonRoom`)的选择。引入 `defaults/catalog/RandomLoadout.boss-names.json` 静态目录文件提供 BOSS 中英文名称映射，旨在消除每次面板重绘时重复加载 `AIActor` 与查询游戏 `StringTableManager` 导致的界面卡顿（注：测试与基线日志系在 Debug 构建版本下采集，Release 版本实际性能更佳）；当 JSON 目录中未查找到对应名称时，自动回退至游戏实时 API 读取。同时新增跳过 BOSS 入场动画(`BossIntroSkipHooks.cs`)与音频诊断(`BossAudioDiagnosticsHooks.cs`)支持。
* 增强战场房间倒放与重置功能（新增玩家状态/属性快照回溯(rewind)与恢复）：在 `src/RandomLoadout/Runtime/RoomEnemyReplayService.cs` 中实现房间倒放时的玩家状态回溯(rewind)机制（`PlayerRewindEnabled`），进入普通战斗房间或 BOSS 房间时自动截取玩家血量、护甲(armor)、空响弹(blanks)、枪械(guns，包括子弹)、主动道具(active items，包含充能状态)与被动道具(passive items)及运行时属性(`stats`)快照。触发倒放时，通过原地原位更新（In-Place Restoration）无缝将玩家恢复至进入房间时的状态，实现零消耗反复练习特定战场；支持快捷键（默认为 `C` 键，可在 `randomgun.randomloadout.cfg` 的 `[UI] RoomEnemyRewindKey` 中自定义；选择 `C` 键系因原版默认未占用，且键鼠玩家左手 WASD、右手鼠标时，无需移动手部位置即可瞬间触发倒放，保障连续战斗体验）或在面板中触发 `Rewind Room` 与 `Respawn Enemies` 操作，配套架构说明详见 `docs/architecture/boss-room-rewind.md`。
* 新增敌人血条显示与伤害诊断功能：新增 `EnemyHealthBarToggleService.cs` 切换敌人血条(Enemy HP Bars)显示，复用游戏内侦测器(Scouter)原生血条样式且支持持久化配置 `[Combat] EnemyHealthBarsEnabled`；新增 `DamageDiagnosticsService.cs` 伤害诊断系统，实时记录并输出玩家伤害结算详情日志。
* 新增角色属性倍率与多项战斗辅助开关持久化/目标指定：在战斗(`Combat`)页面新增玩家伤害 (`1x`, `2x`, `5x`, `10x`, `100x`) 与移动速度 (`1.5x`, `2x`, `3x`) 动态倍率控制 (`PlayerStatMultiplierService.cs`)；重构并增强战斗辅助选项：
  * **配置持久化**：无限弹药(`AmmoModeToggleService`)、自动装弹(`AutoReloadToggleService`)及按住连发(`RapidFireToggleService`)增加配置持久化回调 `_persistMode`/`_persistEnabledState`，状态将自动保存至 `[Combat]` 配置文件并在下次游戏启动时自动恢复。
* 新增双人合作(Co-op)模式全功能玩家目标控制 (Target: `P1`/`P2`/`Both`)：
  * **图鉴物品发放**：在图鉴浏览(`Pickup Browser`)中支持将搜索选择的物品/枪械精确发放给 P1、P2 或双人同时发放(`Both`)。
  * **物资调整与物理掉落生成**：在物品(`Pickups`)页面中，基础资源加给（生命、护甲、弹夹、钥匙、鼠级钥匙、金币）与物理掉落生成(`Spawn`)动作全面支持目标玩家切换，可在指定玩家（P1、P2 或双方）身边生成原生实体拾取物。
  * **角色独立切换与邪教徒支持**：在 `常规(General)` 分类下的 `角色(Characters)` 页面支持单独指定目标玩家（`P1`/`P2`）进行独立替换；同时新增了双人合作模式默认角色 `邪教徒(The Cultist)` 的选择按钮，支持将 P1 或 P2 自由切换为邪教徒或任意其他角色。
  * **战斗辅助独立生效**：无敌模式(`InvincibilityToggleService`)与护甲不消耗(`ArmorNoConsumeToggleService`)重构为按玩家粒度独立管理，可针对指定玩家单独开启防护与 0 护甲预补。

### 技术改进/重构 (Changed/Refactor)
* 重构核心控制逻辑至 Core 纯逻辑层：将光标渲染所有权管理(`CursorRenderOwnershipService`)与玩家输入所有权管理(`PlayerInputOwnershipService`)从 `RandomLoadout` 运行时层提取至 `src/RandomLoadout.Core/` 纯逻辑层，解耦 Unity/ETG 运行时依赖并补充相关单元测试。
* 模块化拆分房间调试服务 (`RoomDebugCommandService`)：将庞大的 `RoomDebugCommandService.cs` 拆分为 `BossSelection.cs` (BOSS 选择)、`RoomReplay.cs` (房间重置与敌人回放)、`Map.cs` (地图传送与开雾)、`Muncher.cs` (枪械吞噬者生成) 与 `Helpers.cs` 等多份责任明确的 partial 文件。
* 优化控制面板(control panel)打开响应与 BOSS 选择渲染性能（注：基线耗时数据均在 Debug 构建版本下测试采集得出，Release 构建版本无调试符号开销，实际运行更流畅）：
  * **控制面板与图鉴长列表启动/渲染优化**：在 Debug 版本下测试，优化前面对 670+ 个物品的图鉴列表，首次构建目录耗时约 289ms，首次列表绘制耗时约 210ms，后续每帧全量绘制耗时约 20–40ms 导致明显卡顿。本版本通过长列表视口虚拟化（仅绘制当前可视区域）、按语言缓存物品目录、按搜索/筛选缓存结果数组以及图标/行控件绘制裁剪，大幅消除了面板打开与滚动时的卡顿。
  * **BOSS 选择列表渲染与名称解析优化**：在 Debug 版本下测试，原版重绘列表时频繁实例化 `AIActor` 及查询游戏 `StringTableManager` 造成明显掉帧。本版本引入 `defaults/catalog/RandomLoadout.boss-names.json` 静态名称字典直接读取中英文名称，消除重复游戏 API 检索开销；当静态字典未命中时自动安全回退至游戏 API 查询。
* 优化控制面板(control panel)导航与 UI 布局：重构战斗(`Combat`)与角色(`Character`)页面的控件排版，优化键盘方向键与手柄 D-pad 的焦点轮转规则；增加实验性功能(Experimental Mode)保护机制，避免未授权误触未成熟工具。
* 补充单元测试与文档索引更新：在 `tests/RandomLoadout.Core.Tests/` 中新增 `CursorRenderOwnershipServiceTests`、`PlayerInputOwnershipServiceTests` 及 `RuntimeHookContractTests` 验证套件；同步更新 `docs/README.md`、`docs/reference/code-index.md`、`docs/reference/commands.md` 与 `docs/reference/config-format.md`。

### 修复 (Fixed)
