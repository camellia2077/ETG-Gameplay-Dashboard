## [v0.3.4] - 2026-06-30

### 新增功能 (Added)
* 通过反编译并复用 ETG 原生资源，补齐了血量、护甲、钥匙、老鼠钥匙、空包弹、弹壳、霸权币等 `拾取物(pickups)` 图标，并将这些图标接入控制面板，使 `pickups` 选择从纯文本改为更直观的“左侧图标 + 名称，右侧操作”展示方式。
* 新增 `Player -> Pickups / 拾取物` 与 `Player -> Stats / 属性` 次级分类，将玩家资源操作与属性类操作拆分显示，并将 `Clear Curse / 清除诅咒` 迁移到 `Stats` 分类。
* 新增 `Armor`、`Blank`、`Key`、`Casings` 的 `No Consume / 不消耗` 开关能力，并为 `Casings / 弹壳` 新增“余额不足时仍允许正常购买、但不实际消耗弹壳”的运行时购买放行逻辑。
* 新增 `EnableCommandPanelCursorVerboseLogs` 调试开关与 `docs/operations/logging-command-panel-cursor.md` 文档，用于排查手柄切换到鼠标后游戏内光标显隐异常的问题。
* 新增 `docs/reference/ui-icon-reuse.md`，记录控制面板复用 ETG 原生拾取物图标资源的实现方式、资源定位来源，以及老鼠宝箱钥匙图标的特殊说明。

### 技术改进/重构 (Changed/Refactor)
* 调整 `Player` 页面布局，将血量、护甲、空包弹、钥匙、老鼠钥匙、弹壳改为“左侧图标 + 名称，右侧操作按钮”的资源行式布局，并按 `Health -> Armor -> Blank -> Key -> Rat Key -> Casings` 顺序统一排列。
* 调整 `General -> Pickups / 拾取物` 页面布局，将原先偏文本式的资源入口改为“左侧图标 + 名称，右侧生成按钮”的资源列表，并允许页面按内容纵向增长而非改为滚动列表。
* 调整 `Player -> Pickups` 的资源操作文案，将血量改为 `恢复 0.5 / 回满 / 增加1上限`，将空包弹改为 `+1 / 不消耗`，并为护甲、钥匙、弹壳补充 `开启不消耗 / 关闭不消耗` 状态型按钮文案与亮暗态区分。
* 调整 `General -> Pickups` 的资源顺序为 `血量、护甲、空包弹、钥匙、老鼠钥匙、弹壳、霸权币`，并将弹壳生成量统一提升为 `+100`。
* 调整 `Start Items / 开局物品` 页面中拾取物图标显示尺寸，使血量、护甲、钥匙、老鼠钥匙、空包弹、弹壳等资源图标在预设详情中更易识别。
* 调整预设详情页相关文本资源，将 `掉落物` 统一更名为 `拾取物`。

### 修复 (Fixed)
* 修复 `General -> Pickups` 页面在手柄 `D-pad` 与键盘方向键下无法选中任何按钮、`B / Delete` 返回也不工作的焦点初始化问题。
* 修复控制面板在打开后进行菜单导航时，会错误触发 ETG 原生 HUD 的“加心 / 加甲”动画重复播放的问题；当前通过隔离输入串入与最大血量/护甲回滚链路，避免仅移动焦点时反复播出资源增长动画。
* 修复 `Player -> Pickups` 中不消耗按钮的状态文案与启用态颜色不一致的问题；未开启时显示 `关闭不消耗` 且保持暗色，开启后显示 `开启不消耗` 且使用亮色按钮样式。
* 修复 `Armor / 护甲` 不消耗在当前护甲为 `0` 时无法建立有效恢复基线的问题；现在开启护甲不消耗时会先补出 `1` 点护甲，以保证后续受伤时能正确走“失去护甲后恢复”的逻辑。
* 修复老鼠钥匙、护甲等拾取物图标映射错误的问题，改为使用已验证的 ETG 原生图标资源，并在代码注释中区分普通钥匙与用于开启复仇老鼠奖励宝箱的 `room_rat_reward_key_001`。
